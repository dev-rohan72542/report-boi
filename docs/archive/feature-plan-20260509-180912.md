# Fix Plan: Sign-Up "Failed to fetch" Error

## Root Cause Analysis

Two issues found:

### Issue 1: Duplicate `NEXT_PUBLIC_SUPABASE_ANON_KEY` in `.env.local`
- Line 5 defines the key with surrounding double-quotes: `NEXT_PUBLIC_SUPABASE_ANON_KEY="eyJ..."`
- Line 11 defines it again without quotes: `NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...`
- Next.js `.env` parsing: when a key is defined twice, the **last definition wins**.
  - Line 11 (unquoted, correct) overrides line 5, so this is NOT the cause.
  - But it's messy and should be cleaned up.

### Issue 2 (PRIMARY CAUSE): `output: 'export'` in `next.config.mjs`
- Static export mode is enabled for Capacitor mobile builds.
- `@supabase/ssr`'s `createBrowserClient` in a statically exported Next.js app running in dev mode can fail to resolve the Supabase URL correctly if the project is **paused on Supabase** (free tier auto-pauses after 1 week of inactivity).
- The `NEXT_PUBLIC_SUPABASE_URL` points to `https://usaidrgkzfoaxxbskgmm.supabase.co` — if this project is paused, all auth requests will fail with a generic "Failed to fetch" network error.

### Issue 3 (SECONDARY): Error message is too generic
- The catch block shows the raw error message, which hides the real problem from the user.
